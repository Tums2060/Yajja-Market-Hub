3. ACCURATE LOCATION SHARING (GPS + Maps Fallback)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Implement robust location capture at checkout with a graceful fallback:

PRIMARY: Use browser Geolocation API (navigator.geolocation.getCurrentPosition)
- Request with high accuracy: { enableHighAccuracy: true, timeout: 10000 }
- On success: reverse-geocode coordinates using Google Maps Geocoding API to get 
  a human-readable address, then pre-fill the delivery address field

FALLBACK (if GPS fails or user denies permission):
- Show a Google Maps embed with a draggable pin centered on Nairobi (default)
- User drags the pin to their exact location
- Show address lookup bar (Places Autocomplete) as an alternative
- Coordinates from the selected map location are captured and stored

STORAGE: Captured coordinates (latitude, longitude) are saved with the order and 
transmitted to the assigned rider. Vendors do NOT receive raw GPS coordinates — 
they only see the human-readable address string.

Show a loading state while location is being captured, and clear error messaging 
if it fails, prompting the user to use the map picker.
